# FlyPrint Edge 直接 IPP 部署与运维

## 环境要求

- Windows 主机与打印机网络可达；网线直连时两端使用同一网段静态 IPv4。
- 打印机启用 IPP 或 AirPrint，并支持直接接收 PDF。
- DOCX 打印需要 LibreOffice；PDF 和图片不依赖 LibreOffice。
- Windows 不需要添加打印机、安装打印驱动或安装 SumatraPDF。

## 首次配置

1. 用网线连接主机和打印机，并确认两端 IP 在同一网段。
2. 在打印机 Web 管理页启用 IPP/AirPrint。
3. 打开 Edge 管理端“打印机管理”。
4. 点击“刷新 IPP 打印机”；若未发现，输入完整 URI，例如 `ipp://192.168.50.2:631/ipp/print` 并检测。
5. 检测兼容后添加打印机并设为默认。
6. 执行“测试打印”；只有设备返回 completed 才算通过。

## 日常判断

- 缺纸、卡纸、机盖打开、离线或耗材耗尽会阻止新任务。
- 碳粉偏低只提示，不阻止任务。
- 打印期间发生明确故障时，Edge 取消设备 Job 并显示错误。
- “无法确认打印结果”表示设备可能已接收任务；不要立即重试。管理员确认打印机中没有遗留任务后，可在管理端解除锁定。

## 地址变化

自动发现再次看到相同 `printer-uuid` 且新 URI 探测通过时，Edge 自动更新保存地址。手工更换地址时，先用完整 URI 检测，再重新添加；不得仅按名称或 IP 判断同一设备。

## 排障顺序

1. 检查网线、电源及主机到打印机 IP 的连通性。
2. 确认打印机 IPP/AirPrint 已启用，URI 资源路径来自设备 DNS-SD `rp` 或设备说明。
3. 在管理端重新检测 URI，查看不兼容原因。
4. 检查 `edge.log` 中的 IPP 操作、request-id、设备 job-id 和状态原因。
5. 故障恢复后确认设备不再报告 error，再恢复二维码和打印。

日志或诊断摘要不得包含 Cloud 密钥、令牌或用户文件内容。

## 升级与卸载

- 从旧版本首次升级到配置版本 2 会清除旧 Windows/WSD/USB 打印机记录，需要重新添加 IPP 设备。
- 工具版本、能力或 URI 改变后应重新执行测试打印。
- 卸载前从托盘退出 Edge；安装程序会停止运行进程后再删除文件。

## 真实设备验收

在网线直连 HP Color LaserJet Pro 3288dn 的目标机器上验证自动发现、手动添加、PDF/DOCX/图片、单双面、彩色/黑白、多份、缺纸、补纸、卡纸、离线、取消、恢复和 Cloud 状态回报。
