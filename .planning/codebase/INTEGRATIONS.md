# External Integrations

**Analysis Date:** 2025-03-17

## APIs & External Services

**Fly-Print-Cloud (Primary Cloud Backend):**
- Purpose: Central cloud service for printer management, job distribution, and monitoring
- Base URL: Configured in `config.json` → `cloud.base_url` (default: http://localhost:8012)
- Auth URL: Configured in `config.json` → `cloud.auth_url`
- Client: `cloud_api_client.py`, `cloud_websocket_client.py`
- Auth: OAuth2 Client Credentials flow via `cloud_auth.py`

**OAuth2 Authentication:**
- Flow: Client Credentials
- Token Endpoint: `cloud.auth_url` (e.g., http://localhost:8012/auth/token)
- Client ID: `config.json` → `cloud.client_id`
- Client Secret: `config.json` → `cloud.client_secret`
- Scopes: `edge:register`, `edge:heartbeat`, `edge:printer`, `edge:print`, `edge:file`
- Token TTL: 1 hour (3600 seconds), auto-refreshed 5 minutes before expiry

**Cloud REST API Endpoints:**
- `POST /api/v1/edge/register` - Register edge node
- `POST /api/v1/edge/{node_id}/printers` - Register printers
- `DELETE /api/v1/edge/{node_id}/printers/{printer_id}` - Delete printer
- `POST /api/v1/edge/{node_id}/printers/status` - Batch status update
- `GET /api/v1/health` - Health check (for latency measurement)

**Cloud WebSocket:**
- URL: Converted from HTTP(S) to WS(S) with `?node_id={node_id}` query param
- Protocol: Bidirectional WebSocket with automatic reconnection
- Ping interval: 30 seconds
- Client: `cloud_websocket_client.py` → `CloudWebSocketClient`

**WebSocket Message Types (Downstream - Cloud → Edge):**
- `print_job` - New print job from cloud
- `preview_file` - File preview request
- `upload_token` - File upload credentials
- `error` - Error messages from cloud
- `server_close` - Server-initiated close (e.g., node_deleted)

**WebSocket Message Types (Upstream - Edge → Cloud):**
- `edge_heartbeat` - System status report
- `submit_print_params` - Print job parameters submission
- `request_upload_token` - Request file upload credentials
- `job_update` - Job status updates (completed/failed)
- `ack` - Message acknowledgments

## Data Storage

**Databases:**
- No database used
- Configuration stored in JSON file: `config.json`

**File Storage:**
- Local filesystem only
- Temporary files: Project-local `temp/` directory (portable mode)
- File management: `file_manager.py` with TTL-based cleanup (default: 30 minutes)
- Preview caching: In-memory with automatic cleanup

**Caching:**
- In-memory preview cache: `preview_cache`, `preview_page_cache` in `main.py`
- File access tokens: In-memory cache (`file_access_tokens`)
- Completed jobs cache: `completed_jobs` in WebSocket client (TTL: 1 hour)

## Authentication & Identity

**Cloud Authentication:**
- OAuth2 Client Credentials flow
- Token storage: In-memory (`CloudAuthClient.access_token`)
- Auto-refresh: 5 minutes before expiry
- Client: `cloud_auth.py`

**Node Identity:**
- Node ID: MAC address without colons (generated by `edge_node_info.py`)
- Cached in: `config.json` → `cloud.node_id`
- Auto-registration: Enabled by `config.json` → `cloud.auto_register`

**File Access Tokens:**
- One-time tokens for downloading files from cloud
- Validity: Short-lived (as specified by cloud)
- Storage: In-memory dictionary (`file_access_tokens` in `main.py`)

## Monitoring & Observability

## Environment Configuration

```json
{
  "cloud": {
    "enabled": true,
    "base_url": "http://cloud-server:8012",
    "auth_url": "http://cloud-server:8012/auth/token",
    "client_id": "edge-default",
    "client_secret": "YOUR_CLIENT_SECRET",
    "node_name": "my-edge-node",

    "auto_register": true
  },

    "bind_address": "127.0.0.1",
    "port": 7860
  }

```

**Secrets Location:**
- `config.json` - Contains client_secret (sensitive)
- Not in version control (listed in `.gitignore`)

## Webhooks & Callbacks

**Incoming (Cloud → Edge):**
- WebSocket messages (real-time)
- No HTTP webhooks

**Outgoing (Edge → Cloud):**
- WebSocket messages (bidirectional)
- HTTP REST API calls for registration and status updates

**SSE (Server-Sent Events):**
- Endpoint: `/api/events`
- Purpose: Push cloud messages to frontend UI
- Event types: `preview_file`, `error`, `cloud_error`, `job_status`, `node_status_changed`

## Network Printer Discovery

**mDNS/Bonjour Discovery:**
- Library: `zeroconf`
- Service type: `_ipp._tcp.local.`
- Used for auto-discovering network printers
- Configurable: Can use static list instead (`config.json` → `printers.discovery_mode`)

## Document Conversion

**Word to PDF:**
- Windows: COM automation via `win32com.client`
- Supported applications: WPS Office (`Kwps.Application`), Microsoft Word (`Word.Application`), LibreOffice
- Linux: LibreOffice command-line (`soffice --headless --convert-to pdf`)
- Temporary output to portable temp directory
